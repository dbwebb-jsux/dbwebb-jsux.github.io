Vi kommer igenom kursen ta ett analytiskt perspektiv på en verklig webbplats design utifrån temat från kursen: Att bygga en webbshop. I detta kursmoment tar vi ett lite mer tekniskt perspektiv.



## Skapa rapportfilen

Skapa en struktur för din rapport, det kommer fler sådana här uppgifter.

```bash
# Stå i kursrepot
touch reports/kmom04.md
```


## Rapportstruktur

När du skriver din rapport så kan du följa denna rubrikstruktur. Det är en struktur som är vanligt använd i rapportsammanhang (och i ditt kommande examensarbete).

```text
Titel på rapporten
=======================

Skriv en eller två rader om vad uppgiften handlar om.

Urval
-----------------------

Berätta vilken webbplats du valt att undersöka och varför eller hur du gick tillväga när du gjorde ditt urval.

Metod
-----------------------

Berätta kort om din "metod", hur du gör för att utföra undersökningen. Berätta om du använder något speciellt verktyg.

Resultat
-----------------------

Dokumentera dina resultat från din studie. Berätta vad du kom fram till, vilka resultat du hittade och observerade.

Denna veckan kan en tabell i markdown vara på sin plats.

|               | Antal filer  | Mängd JavaScript  | Laddningstid  |
|---------------|--------------|-------------------|---------------|
| Förstasidan   |              |                   |               |
| Produktsidan  |              |                   |               |
| Ordersida     |              |                   |               | 



Analys
-----------------------

Diskutera och analysera de resultaten du fann.

Referenser
-----------------------

Ange de eventuella referenser du använder dig av, om några.
```

Strukturen är inte absolut nödvändig, se den som en möjlighet att träna på god struktur av rapportskrivning.



Krav
-----------------------

1. Vi ska nu välja ytterligare 2 webbshoppar att analysera. 

1. Utifrån dina valda webbshoppar ska vi analysera hur mycket resurser som laddas in och används på webbplatsen.

1. Välj tre liknande enskilda sidor från dina webbshoppar, förslagsvis förstasidan, en produktsida och en ordersida/varukorgen.

1. Använd Network-fliken i din webbläsare och ladda om sidan utan cache.

1. För varje sida för varje webbshop notera:
    1. Hur många filer laddas in på respektive sida?
    1. Hur stor mängd data laddas på respektive sida?
    1. Hur lång tid tar det att ladda respektive sidorna?

1. Reflektera ur ett hållbarhetsperspektiv kring webbshopparnas resurser.
